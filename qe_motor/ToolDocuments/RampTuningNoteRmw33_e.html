Notes on Tuning in Projects Using RAMP

Notes on Launching the Tuner

When using Renesas Motor Workbench version 3.3 or earlier, in projects that use the motor control FSP modules of RAMP (Renesas Advanced Motor control Platform), the Tuner cannot be launched directly from the “Launch Renesas Motor Workbench tuner” button in QE for Motor.

Please follow the procedure below to launch the Analyzer first, and then switch to the Tuner.

  1. After writing the program used for tuning, open the “Analyzer” page in QE for Motor, and click the “Launch Renesas Motor Workbench analyzer” button to launch the Analyzer of Renesas Motor Workbench.
  2. In the Analyzer view of Renesas Motor Workbench, click the “Tuner” button to switch to the Tuner.

Notes on Applying Tuning Results to FSP Settings

In projects that use the motor control FSP modules of RAMP (Renesas Advanced Motor control Platform), when using the function to apply tuning results to FSP settings via the “Apply the tuning results to the target project” button in QE for Motor, please note the following points.

  • After completing tuning in Renesas Motor Workbench, specify a location within the e2 studio project folder as the output destination when exporting the header files containing the tuning result parameter values by using the “Output Header File” button. The relevant header files are r_mtr_control_parameter.h and r_mtr_motor_parameter.h.
  • In the RAMP sample project (RA6T2_MCB2_MCILV1_RAMP_E2S_V100), specify the following folder as the output destination and overwrite the existing header files. 
    RA6T2_MCB2_MCILV1_RAMP_E2S_V100\src\application\main

    In the initial state of the sample project, these header files are not used by the program. However, if you modify the program to use these header files, be sure to create a backup in advance as necessary.
