RAMPを使用したプロジェクトでチューニングする際の注意事項

Tunerの起動に関する注意事項

Renesas Motor Workbench 3.3 以下を使用している場合、 RAMP (Renesas Advanced Motor control Platform) のモータ制御用 FSP モジュールを使用したプロジェクトでは、 QE for Motor の「Renesas Motor Workbench tuner を起動」ボタンから、 Tuner を直接起動することができません。

以下の方法で、一旦 Analyzer を起動後、Tuner に切り替えてください。

  1. チューニングに使用するプログラムを書き込み後、 QE for Motor の「アナライズ」ページを表示し、 「Renesas Motor Workbench analyzer を起動」ボタンで、 Renesas Motor Workbench の Analyzer を起動する。
  2. Renesas Motor Workbench の Analyzer 表示で 「Tuner」ボタンをクリックし、Tuner に切り替える。

チューニング結果のFSP設定への適用に関する注意事項

RAMP (Renesas Advanced Motor control Platform) のモータ制御用 FSP モジュールを使用したプロジェクトでは、 QE for Motor の「チューニング結果をプロジェクトに適用する」ボタンにより チューニング結果を FSP 設定に反映する機能を使用する場合、以下の点に注意してください。

  • Renesas Motor Workbench でのチューニング完了後、 「Output Header File」ボタンでチューニング結果のパラメータ値を含む ヘッダーファイル（r_mtr_control_parameter.h、 r_mtr_motor_parameter.h）の出力先に、 使用している e2 studio プロジェクトフォルダ内を指定してください。
  • RAMP 使用サンプルプロジェクト （RA6T2_MCB2_MCILV1_RAMP_E2S_V100）では、 出力先に以下のフォルダを指定し、既存のヘッダーファイルを上書きしてください。 
    RA6T2_MCB2_MCILV1_RAMP_E2S_V100\src\application\main

    サンプルプロジェクトの初期状態では、これらのヘッダーファイルはプログラムで使用されていませんが、 使用するようプログラムを変更された場合は、必要に応じて事前にバックアップを保存してください。
